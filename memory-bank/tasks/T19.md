# T19: Cross-Platform / OS-Agnostic Fixes

**Status:** ✅ COMPLETE
**Completed:** 2026-06-24
**Type:** Technical Debt / Infrastructure

## Problem

The codebase was originally developed on macOS and contained multiple macOS-specific assumptions that caused errors or misleading behavior when run on Linux (e.g., a VPS):

1. `SleepWakeDetector.ts` called `pmset` and `ioreg` unconditionally — both macOS-only tools
2. `/api/restart` endpoint had a hardcoded `/Users/sage/...` path
3. `ignoredProcesses` only listed macOS kernel processes, causing false spike alerts on Linux
4. Dashboard battery UI showed `0%` on battery-less machines, looking broken

## Solution

### 1. SleepWakeDetector.ts — Platform Detection
- Added `getPlatform()` helper: returns `'darwin' | 'linux' | 'windows' | 'other'`
- **macOS** (`darwin`): unchanged — `ioreg` + `pmset` for sleep/wake + battery
- **Linux** (`linux`): reads `/sys/class/power_supply/BAT*/capacity` and `status` for battery; sleep detection returns `"awake"` (servers don't sleep)
- **Other**: graceful no-op with single log warning
- **Commit:** `1561ca2`

### 2. web/server.ts — /api/restart Path
- **Before:** `cd /Users/sage/.openclaw/workspace/code/process-monitor && bash run.sh`
- **After:** `cd "${process.cwd()}" && nohup npx tsx src/main.ts ...` with `HOME: process.env.HOME || '/tmp'`
- **Commit:** `18a7024`

### 3. ConfigManager.ts — Ignored Processes
- **Before:** `['kernel_task', 'WindowServer', 'mds', 'mdworker']`
- **After:** added Linux kernel threads: `kworker`, `ksoftirqd`, `rcu_preempt`, `migration`, `watchdogd`, `cpuhp`, `khugepaged`, `kcompactd0`, `oom_reaper`
- Prevents false spike alerts from Linux kernel threads
- **Commit:** `18a7024`

### 4. web/public/app.js — Battery UI
- **Before:** always showed `0%` + "Plugged In" on battery-less machines
- **After:** detects `percent === 0 && isPlugged` as "no battery", displays `— N/A` / "No battery" / "Desktop / Server"
- **Commit:** `18a7024`
