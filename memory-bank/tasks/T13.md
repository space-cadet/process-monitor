# T13: Process Tree View (Hierarchical)

*Created: 2026-06-18*
*Status: ⬜ PENDING*
*Priority: MEDIUM*

## Problem
Flat process lists are misleading. `Code Helper (Renderer)` appears as a separate process from `Visual Studio Code`, when it's actually VS Code's child. Users can't tell which *app* is responsible for CPU usage.

## Solution
Show process hierarchy: parent → children. Aggregate child CPU/memory into parent totals.

```
Visual Studio Code (PID 1234) — 35% CPU, 800 MB
  ├─ Code Helper (Renderer) — 12% CPU, 300 MB
  ├─ Code Helper (GPU) — 8% CPU, 150 MB
  └─ Code Helper (Extension Host) — 15% CPU, 350 MB

Google Chrome (PID 5678) — 28% CPU, 1200 MB
  ├─ Chrome Helper (Renderer) — 10% CPU, 400 MB
  ├─ Chrome Helper (Renderer) — 8% CPU, 350 MB
  └─ Chrome Helper (GPU) — 10% CPU, 450 MB
```

## Technical Approach

### Data Collection
- `systeminformation.processes()` already returns `ppid` (parent PID)
- Build tree from `pid` → `ppid` relationships
- Aggregate: sum child CPU% and memory into parent

### Tree Algorithm
```typescript
function buildProcessTree(processes: ProcessSnapshot[]): ProcessTreeNode[] {
  const byPid = new Map(processes.map(p => [p.pid, p]));
  const children = new Map<number, ProcessSnapshot[]>();
  
  for (const proc of processes) {
    const parent = byPid.get(proc.ppid);
    if (parent) {
      children.set(parent.pid, [...(children.get(parent.pid) || []), proc]);
    }
  }
  
  // Return root-level processes with nested children
}
```

### Dashboard UI
- Toggle: "Flat View / Tree View"
- Tree: collapsible, indented, with aggregate bars
- Click parent → expands children
