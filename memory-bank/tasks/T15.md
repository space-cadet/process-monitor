# T15: macOS Energy API Integration

*Created: 2026-06-18*
*Completed: 2026-06-24*
*Status: ✅ COMPLETE*

## What Was Built

### EnergyCollector (`src/core/EnergyCollector.ts`)
- Integrates `powermetrics` (macOS) for per-process energy data
- Runs `powermetrics --samplers tasks -n 1 -i 1000` periodically
- Parses output to extract per-process energy in millijoules (mJ)
- Gracefully handles missing `sudo` — falls back to null energy values

### Database
- `energy_mj REAL` field added to `process_samples` table
- Stores per-process energy impact alongside CPU/memory data

### Dashboard
- Energy displayed in **process cards** when available
- Shows `⚡ N mJ` badge next to process name
- Hidden when energy data is unavailable (no `sudo`)

### Limitations
- **`powermetrics` requires `sudo`** — energy data only available when monitor runs with privileges
- **macOS-only** — Linux/Windows don't have equivalent unified energy score
- **Sampling overhead** — `powermetrics` adds ~1-2% CPU overhead during sample
- **Fallback**: When energy is unavailable, approximate via `cpu_percent * 0.6 + memory_percent * 0.2`
