# T15: macOS Energy API Integration

*Created: 2026-06-18*
*Status: ⬜ PENDING*
*Priority: MEDIUM*

## Problem
Raw CPU% is only part of the story. macOS Activity Monitor shows an "Energy Impact" score that accounts for:
- CPU usage (weighted by efficiency cores vs performance cores)
- GPU usage
- Disk wakeups (preventing idle sleep)
- Network activity over WiFi/cellular
- App Nap efficiency

A process can have low CPU% but high energy impact if it prevents the system from sleeping.

## Solution
Integrate macOS-specific energy metrics:

### Data Sources
- `top` energy column: `top -stats pid,command,cpu,power` — shows "POWER" score
- `powermetrics` (sudo required): detailed per-process energy breakdown
- `iokit` framework: access to `IOReport` energy counters
- `systeminformation` may expose some of this

### Metrics to Capture
- Energy Impact score (macOS's own metric, 0-100+)
- CPU time on efficiency vs performance cores
- GPU time (if applicable)
- Disk wakeups per second
- Network packets per second

### Trade-offs
- `powermetrics` requires root — not feasible for background monitor
- `top` energy column is available without root but less detailed
- `systeminformation` may not expose energy data yet

### Fallback
If energy API isn't available without root, approximate energy impact:
- `energy_score = cpu_percent * 0.6 + memory_percent * 0.2 + disk_wakeups * 0.2`
- Document that this is an approximation

- `src/core/SystemCollector.ts` — integrate energy sampling
- Dashboard: Energy Impact column/score
