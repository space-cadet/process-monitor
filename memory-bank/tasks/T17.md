## What Was Built

### V1 Scope (Completed)
- Device identity (UUIDv4 `did`)
- QR code pairing

- Device registry with online/offline status

## Components

### 1. DeviceIdentity (`src/core/DeviceIdentity.ts`)
- Generates UUIDv4 `did` on first boot
- Persists to `~/.procmon/config/device.json`
- Provides `getIdentity()`, `getDid()`, `getName()`

### 2. DeviceRegistry (`src/core/DeviceRegistry.ts`)
- JSON file storage: `~/.procmon/devices.json`
- CRUD operations: `register()`, `heartbeat()`, `list()`, `remove()`
- Auto-updates `lastSeen` on heartbeat
- Stores `endpoint` (metrics URL) per device

### 3. Network Detection (`src/web/server.ts`)
- `getTailscaleIP()` — detects `100.x.x.x` from `utun` interface
- `getLanIP()` — detects `192.168.x.x` / `10.x.x.x` / `172.16-31.x.x`
- `getAllHosts()` — returns all available networks

### 4. API Endpoints
- `GET /api/identity` — returns device info + endpoints for all networks
- `GET /api/qr` — SVG QR code with identity JSON
- `GET /api/metrics?since=ISO&limit=N` — peer metrics endpoint
- `GET /api/devices` — list registered devices
- `POST /api/devices/register` — register new device
- `POST /api/devices/heartbeat` — device heartbeat
- `DELETE /api/devices?id=X` — remove device

### 5. Dashboard UI
- **Devices tab** — online/offline status cards, battery sparklines
- **"Show QR" button** — opens modal with scannable QR code
- **Peer polling** — fetches metrics from registered devices every 30s
- **Network badges** — shows which networks each device is reachable on

## Network Modes

| Mode | IP | Use Case | Requires |
|------|-----|----------|----------|
| Local | `localhost:3456` | Same machine | Nothing |
| LAN | `192.168.1.42:3456` | Same WiFi | Same network |
| Tailscale | `100.92.54.38:3456` | Anywhere | Tailscale daemon |

## Limitations
- **Android VPN conflict**: Tailscale and NordVPN can't run simultaneously on Android
- **LAN-only**: Works for same-network monitoring when Tailscale unavailable
- **No relay**: Direct P2P only — no cross-network without Tailscale
- **No mDNS**: V1 uses manual QR scan or IP entry (no auto-discovery)
- **No authentication**: V1 relies on network-level security
