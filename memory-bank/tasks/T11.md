# T11: Natural Language Search / Query Box

*Created: 2026-06-18*
*Status: ⬜ PENDING*
*Priority: MEDIUM*

## Problem
The dashboard shows current state and basic history. But answering questions like "Show me all processes >20% CPU yesterday between 2-4pm" or "When did battery drop >10% in 30 min last week?" requires writing SQL or clicking through charts.

## Solution
A search box in the dashboard that accepts natural language queries and translates them to SQL:

| User Input | SQL Equivalent |
|-----------|----------------|
| "Chrome CPU > 30% yesterday" | `SELECT * FROM process_samples WHERE name LIKE '%Chrome%' AND cpu_percent > 30 AND timestamp > now - 86400000` |
| "battery dropped 10% in 30 min" | Drain events with drop >= 10 and duration <= 30 |
| "top 5 memory hogs this week" | Aggregate RSS by process over 7 days |
| "when did Docker spike" | `SELECT * FROM process_spikes WHERE name LIKE '%Docker%'` |

## Technical Approach

### Query Parser
Simple rule-based parser (no LLM needed for this scope):
1. Tokenize: time expressions ("yesterday", "2 hours ago", "last week"), metrics ("CPU", "memory", "battery"), operators (">", "<", "top"), process names
2. Map tokens to SQL fragments
3. Execute and format results

### Time Expression Parser
- "yesterday" → `timestamp > now - 86400000`
- "last 2 hours" → `timestamp > now - 7200000`
- "between 2pm and 4pm" → parse to timestamps
- "this week" → since Monday 00:00

### Result Rendering
- Table for process queries
- Mini chart for time-series queries
- Summary card for aggregate queries
