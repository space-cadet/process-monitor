# T8: LaunchDaemon Installation for Auto-Start

*Created: 2026-06-13 02:34 IST*
*Completed: 2026-06-15 06:27 IST*
*Status: ✅ COMPLETE*

## Overview

Install mac-process-monitor and its web dashboard as macOS LaunchDaemons so they auto-start on boot without requiring a user login.

## Background

The monitor was running as a foreground process (`npx tsx src/main.ts`) and the dashboard as a one-off server. Both needed to be restarted manually after reboots. The LaunchDaemon pattern was proven with `ai.openclaw.gateway.sage` (T31), so the same approach was applied here.

## Implementation

### Two LaunchDaemons Created

1. **`ai.openclaw.procmon.monitor`** — Core monitor loop
   - Runs `npx tsx src/main.ts` as `sage` user
   - Working directory: `~/.openclaw/workspace/code/mac-process-monitor`
   - Logs to `~/Library/Logs/openclaw/procmon.log`

2. **`ai.openclaw.procmon.dashboard`** — Web dashboard server
   - Runs `npx tsx src/dashboard.ts` as `sage` user
   - Working directory: `~/.openclaw/workspace/code/mac-process-monitor`
   - Serves on `http://localhost:3456`
   - Logs to `~/Library/Logs/openclaw/dashboard.log`

### Installation Commands (run as `deepak` with sudo)

```bash
sudo mv ~/.openclaw/workspace/code/mac-process-monitor/ai.openclaw.procmon.monitor.plist /Library/LaunchDaemons/
sudo mv ~/.openclaw/workspace/code/mac-process-monitor/ai.openclaw.procmon.dashboard.plist /Library/LaunchDaemons/
sudo chown root:wheel /Library/LaunchDaemons/ai.openclaw.procmon.*
sudo chmod 644 /Library/LaunchDaemons/ai.openclaw.procmon.*
sudo launchctl bootstrap system /Library/LaunchDaemons/ai.openclaw.procmon.monitor.plist
sudo launchctl bootstrap system /Library/LaunchDaemons/ai.openclaw.procmon.dashboard.plist
```

### Issue Resolved: Port Conflict

**Problem**: Dashboard kept crashing with `EADDRINUSE` because an old `com.mac-process-monitor.dashboard.plist` was still running from a previous setup, holding port 3456.

**Root cause**: An earlier version of the dashboard plist was installed under a different label (`com.mac-process-monitor.dashboard`) and was never removed. When the new `ai.openclaw.procmon.dashboard` tried to bind to port 3456, it failed.

**Solution**:
1. Unloaded the old plist: `sudo launchctl bootout system com.mac-process-monitor.dashboard`
2. Removed the old plist file: `sudo rm /Library/LaunchDaemons/com.mac-process-monitor.dashboard.plist`
3. Reloaded the new plist: `sudo launchctl bootstrap system /Library/LaunchDaemons/ai.openclaw.procmon.dashboard.plist`

**Verification**: Both services running:
- `ai.openclaw.procmon.monitor` → PID 53211, status 0
- `ai.openclaw.procmon.dashboard` → PID 57453, status 0
- Dashboard responding with HTTP 200 on http://localhost:3456
