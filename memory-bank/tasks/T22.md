## Description
The current monitor is useful for sustained CPU, memory, battery, spike, dashboard, and process-tree visibility, but it does not yet fully identify troublesome processes. It needs a forensic layer that can answer what a process is, who launched it, whether it is stale or orphaned, which service owns it, which files or sockets it has open, and what it is doing during a suspicious interval.

## Acceptance Criteria
- [x] Add a portable interval CPU profiler that records cumulative CPU seconds per process over a chosen window, not only instantaneous percent CPU.
- [x] Add a live process identity model: PID, PPID, UID/user, executable path, command line, elapsed time, parent chain, state, RSS, and cumulative CPU time.
- [ ] Store enough identity/provenance data in SQLite to distinguish similarly named processes such as `node`, app helpers, browser renderers, and OpenClaw/Kimi/Sage services.
- [ ] Add a macOS forensic adapter for launchd labels, LaunchAgent/LaunchDaemon plist validation, missing program targets, listener ports/open files, and on-demand stack/IO tracing. Initial `lsof` open-file/listener evidence is implemented; launchd/plist/sample/fs_usage remain follow-up.
- [ ] Keep the architecture cross-platform by defining a common evidence model with macOS, Linux, and Windows adapters.
- [ ] Document Linux and Windows equivalents without blocking the macOS-first implementation.
