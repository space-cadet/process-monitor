# T9: Sleep/Wake Correlation Tracking

*Created: 2026-06-18*
*Completed: 2026-06-24*
*Status: ✅ COMPLETE*

## What Was Built

### Database
- `sleep_wake_events` table:
  - `id INTEGER PRIMARY KEY AUTOINCREMENT`
  - `timestamp INTEGER NOT NULL`
  - `event_type TEXT NOT NULL` — 'sleep' or 'wake'
  - `battery_percent REAL`, `duration_minutes REAL`

### API
- `GET /api/sleep-wake-events?since=24h|7d` — returns sleep/wake events with battery levels

### Dashboard
- **Sleep tab** in dashboard — shows timeline of sleep/wake events
- Displays event type, timestamp, battery % at event time
- Shows duration for completed sleep periods

## Files
- `src/web/server.ts` — `/api/sleep-wake-events` endpoint (added to existing server)
- `web/public/app.js` — Sleep tab rendering
- `web/public/index.html` — Sleep tab UI
- `web/public/styles.css` — Sleep tab styles

## Limitations
- Sleep/wake detection uses `pmset` / IOKit notifications — requires macOS
- Events are logged when the monitor is running; sleep events during monitor downtime are missed
- No process correlation during sleep (processes don't sample while asleep)
