# T9: Sleep/Wake Correlation Tracking

*Created: 2026-06-18*
*Status: ⬜ PENDING*
*Priority: HIGH*

## Problem
When a MacBook sleeps, processes can still run (Power Nap, background tasks, Bluetooth wakeups). Users often find their battery dropped 20% while "asleep" with no idea why. Current monitor only samples when awake.

## Solution
Log sleep/wake events and correlate them with battery changes. Show a timeline: "Slept at 23:00 (87%), woke at 07:00 (62%) — 25% drain during sleep. Top wake events: 3x Chrome network activity, 1x Time Machine backup."

## Technical Approach

### Data Collection
- macOS emits `com.apple.powersources.source` notifications via `pmset` / IOKit
- Alternative: parse `log show --predicate 'eventMessage contains "Sleep" OR eventMessage contains "Wake"'`
- Store in new `sleep_wake_events` table:
  - `timestamp`, `type` (sleep/wake), `battery_percent`, `duration_minutes` (for wake: time since last sleep)

### Analysis
- On wake: compare battery % before sleep vs after wake
- Query process samples that occurred during sleep window (if any were captured — some Macs sample during Power Nap)
- Flag anomalous sleep drain (>5% overnight = investigation worthy)

### Dashboard UI
- New "Sleep" tab or panel in dashboard
- Timeline: sleep blocks shown as grey bars, wake events as markers
- Per-sleep report: duration, drain %, suspected culprits
