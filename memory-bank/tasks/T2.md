# T2: Telegram/OpenClaw Alert Integration

*Created: 2026-05-18*
*Last Updated: 2026-06-18*
*Status: ✅ COMPLETE*

## Description
Wire the monitor's alert system to send real notifications instead of just console logs.

## Implementation

### Files Added
- `src/core/AlertSender.ts` — Unified alert dispatcher supporting:
  - Telegram Bot API (via `fetch` to `api.telegram.org`)
  - macOS native notifications (via `osascript`)
  - Per-event cooldowns (drain: configurable, spike: 5min per-process)

### Files Modified
- `src/core/Monitor.ts` — Replaced `sendAlert()` stub with `AlertSender` integration
  - `handleDrainEvent()` → `alertSender.sendDrainAlert()`
  - `handleSpike()` → `alertSender.sendSpikeAlert()`
  - `handleBatteryImpactEvent()` → `alertSender.sendBatteryImpactAlert()`

### Configuration
Uses existing `AlertConfig.telegramBotToken` and `telegramChatId` fields in `config.json`.

### Alert Format
- **Drain**: `⚠️ Rapid Battery Drain` — battery % drop, rate, duration, top processes
- **Spike**: `🔥 ProcessName CPU Spike` — value vs baseline
- **Battery Impact**: `🔋 Battery Impact Period` — drop %, duration, top impacts

### Fallbacks
1. Telegram configured → sends Telegram + macOS notification
2. Telegram not configured → macOS notification only
3. Not on macOS → Telegram only (or nothing if no token)
