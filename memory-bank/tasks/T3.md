## Details

Build a query interface to inspect per-process resource usage over time. Users can ask "What was Chrome doing between 2-3pm?" and get CPU/memory timelines for specific processes.

## Progress
1. ⬜ Add process-centric queries to TimeSeriesDB
2. ⬜ Build CLI query tool (e.g., `npx tsx src/query.ts --process Chrome --since 2h`)
3. ⬜ Add aggregation: avg CPU, peak CPU, total runtime
4. ⬜ Add correlation: which processes spiked during drain events?
5. ⬜ Consider web UI for visual timelines

## Files
- `src/storage/TimeSeriesDB.ts` - Add getProcessHistory(), getDrainCorrelations()
- `src/query.ts` - CLI query tool
- `src/utils/formatters.ts` - Table/chart formatting

## Decisions
- SQLite stores top 20 processes per snapshot (already implemented)
- Full history for "flagged" processes (those that correlate with drain)
- Normal processes rolled off after 24h to manage DB size
- Query language: simple CLI flags for now, SQL for power users

## Dependencies
- T1 (completed) — DB schema and sampling must exist
- T2 (pending) — Drain events must be recorded for correlation queries
