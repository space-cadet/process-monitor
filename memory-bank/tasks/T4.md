## Details

Build a lightweight web dashboard showing live battery, CPU, and process data. Serves from the monitor process or as a separate dev server.

## Progress
1. ✅ Add HTTP server (`src/web/server.ts`) — Native Node HTTP server on port 3456
2. ✅ Build HTML dashboard (`web/public/index.html`) — Dark theme, KPI cards, process table
3. ✅ Frontend logic (`web/public/app.js`) — Live data fetching, chart rendering, sorting
4. ✅ Styling (`web/public/styles.css`) — Dark theme, responsive, CPU bars
5. ✅ Auto-refresh every 5 seconds
6. ✅ Battery History chart with time-range buttons (1h/6h/24h)
7. ✅ Drain Events panel with CSV export
8. ✅ Accessible from local network (0.0.0.0:3456)

## Fixes Applied (2026-05-25)
- Fixed chart click handlers (`event.target` undefined bug in `sortProcesses`/`loadHistory`)
- Fixed `renderChart` to handle snake_case DB column names (`battery_percent` vs `batteryPercent`)
- Fixed server to use live collection for `/api/snapshot` instead of stale DB cache
- Process memory values now display correctly in MB

## Files
- `src/web/server.ts` - Native Node HTTP server + API routes

- `web/public/styles.css` - Dark theme styling

## Decisions
- Keep dashboard as separate dev server (`pnpm run dashboard`)
- Use vanilla JS + CSS bars (no Chart.js dependency)
- Port 3456 (same as chimera-chat demo for consistency)
- No auth needed (local only)
- Server binds to 0.0.0.0 for local network access
