## Decisions
- **TypeScript over Python** — Type safety, easier Swift port, better async handling, cross-platform
- **systeminformation library** — Cross-platform, comprehensive, well-maintained
- **better-sqlite3** — Fast synchronous SQLite for time-series data
- **No UI yet** — Console output for now; web dashboard or menubar app later
- **CPU total via si.currentLoad()** — Uses `currentLoad` (camelCase) with `avgLoad` fallback
- **WAL mode** — Enables concurrent reads for dashboard (T4) without write locks
- **Auto-migration** — ALTER TABLE for missing columns preserves data across schema changes

## Known Issues
- `si.battery().timeRemaining` and `.cycleCount` null on some macOS versions
- Process CPU from systeminformation is point-in-time, not averaged over interval
- PID reuse means process identity across snapshots is name-based heuristic, not guaranteed
- Periodic cleanup uses 1% random chance per tick — not deterministic, should be scheduled
- `sendAlert()` formats messages but doesn't dispatch yet (T2)
- No configuration file — all settings hardcoded in `main.ts`
